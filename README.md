1. Requirements Analysis
The system is designed to manage student academic results.
It allows users to add student details and view results.

Functional Requirements:
- Add student name
- Add course
- Add score
- View all results

Non-Functional Requirements:
- Easy to use
- Reliable
- Fast

2. System Design
The system is a console-based Python application.
It uses a dictionary to store student records.

3. Implementation
The project was implemented using Python due to its simplicity and readability.

4. Testing
The system was tested by adding multiple records and viewing stored results.
All functions worked correctly.

5. Deployment
The system was deployed by running the Python file on a local computer.

6. Maintenance
Future improvements include database integration and graphical user interface.
